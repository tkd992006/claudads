User ↔ Advertiser 데이터 관계도

관계 요약 (한 줄)

User 1 ↔ 0..1 AdvertiserAccount 1 ↔ N Ad. 즉, 한 유저는 광고주 계정을 0개 또는 1개만 가질 수 있고, 광고주 계정 하나는 여러 광고를 가질 수 있음.

User

테이블: "User"
  • id (PK, cuid)
  • githubId (unique)
  • login, email, avatarUrl
  • role: VIEWER | ADMIN
  • banned
── 1:0..1 ──▶
User.id = AdvertiserAccount.userId

AdvertiserAccount

테이블: "AdvertiserAccount"
  • id (PK, cuid)
  • userId (unique → User.id)
  • name
  • balanceCents (잔액, 충전식)

AdvertiserAccount

위와 동일
── 1:N ──▶
AdvertiserAccount.id = Ad.advertiserId

Ad

테이블: "Ad"
  • id (PK)
  • advertiserId (FK)
  • title, videoKey, durationSec
  • cpmCents, budgetCapCents, spentCents
  • status: PENDING/APPROVED/REJECTED/PAUSED/EXHAUSTED

핵심 흐름

  1. GitHub OAuth로 로그인하면 User 행이 생김 (web/lib/auth.ts:25 upsert).
  2. 유저가 /advertiser에서 “광고주 계정 만들기” 클릭 → POST /api/advertiser/account가 AdvertiserAccount 생성 (web/app/api/advertiser/account/route.ts).
  3. /advertiser/billing에서 잔액 충전 → balanceCents 증가.
  4. /advertiser/ads/new에서 광고 등록 → Ad 생성, 상태 PENDING.
  5. 관리자(/admin/ads)가 검수 → APPROVED / REJECTED.

중요한 제약

  • AdvertiserAccount.userId가 UNIQUE이므로 한 유저가 광고주 계정을 두 개 만들 수 없음.
  • User.role은 광고주 자격과 무관 — ADMIN이 아닌 일반 VIEWER도 광고주 계정을 만들 수 있음.
  • 광고주 계정 삭제 전에 Ad 행들을 먼저 정리해야 함 (FK 때문).

3) psql (터미널)

# 접속
psql "postgres://postgres:postgres@localhost:5432/flaude"

# 전체 유저 + 광고주 계정 조인 조회
SELECT u.login, u.role, a.name AS advertiser_name, a."balanceCents"
FROM "User" u
LEFT JOIN "AdvertiserAccount" a ON a."userId" = u.id
ORDER BY u."createdAt" DESC;

# 잔액 충전 (수동, cents 단위)
UPDATE "AdvertiserAccount"
SET "balanceCents" = "balanceCents" + 1000000  -- $10,000.00
WHERE "userId" = (SELECT id FROM "User" WHERE login = 'tkd992006');

# 광고를 강제 APPROVED 로
UPDATE "Ad" SET status = 'APPROVED' WHERE id = '광고id';

# 유저에게 ADMIN 역할 직접 부여
UPDATE "User" SET role = 'ADMIN' WHERE login = 'tkd992006';

# 광고주 계정 삭제 (먼저 Ad들 지워야 함)
DELETE FROM "Ad" WHERE "advertiserId" = (
  SELECT id FROM "AdvertiserAccount" WHERE "userId" = '유저id'
);
DELETE FROM "AdvertiserAccount" WHERE "userId" = '유저id';

⚠️ PostgreSQL은 식별자 대소문자 보존을 위해 "User", "AdvertiserAccount"처럼 큰따옴표 필수. balanceCents, userId 같은 컬럼도 큰따옴표로 감싸야 함.
